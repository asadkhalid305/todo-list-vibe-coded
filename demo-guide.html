üß™ Testing Scenarios

1. Basic Functionality

  1. Add a new task by typing and clicking "Add"
  2. Mark a task as complete by clicking the checkbox
  3. Delete a task using the red delete button
  4. Test filtering with "All", "Pending", "Completed" tabs

2. Responsive Design

  1. Open browser dev tools (F12)
  2. Toggle device simulation
  3. Test various screen sizes (iPhone, iPad, Desktop)
  4. Verify touch targets are appropriate for mobile

3. Persistence & Sync

  1. Add several tasks
  2. Refresh the page - tasks should persist
  3. Open the app in another tab
  4. Make changes in one tab - should sync to the other

4. Accessibility

  1. Navigate using only the keyboard (Tab, Enter, Space)
  2. Test with a screen reader if available
  3. Check focus indicators are visible
  4. Enable high contrast mode in OS settings

5. Edge Cases

  1. Try adding empty tasks (should show validation)
  2. Add very long task text (should handle gracefully)
  3. Add many tasks (50+) to test performance
  4. Test with disabled JavaScript (graceful degradation)
